# 操作日志功能优化计划

## 一、现状分析

### 1.1 已有功能

项目中**已经存在完整的操作日志功能**：

| 模块 | 文件 | 说明 |
|------|------|------|
| 数据库表 | `sys_oper_log` | 操作日志记录表（已存在） |
| 后端注解 | `@Log` | 自定义操作日志注解（`ciyo-common`模块） |
| AOP切面 | `LogAspect` | 自动拦截记录操作日志 |
| Service | `SysOperLogService` | 操作日志服务 |
| Mapper | `SysOperLogMapper` | 数据访问层 |
| Controller | `SysOperlogController` | 操作日志API（`/monitor/operlog`） |
| 前端页面 | `views/system/operlog/index.vue` | 操作日志列表页 |
| API接口 | `api/system/operlog.ts` | 前端接口调用 |

### 1.2 已有注解使用情况

全项目共有 **142处** `@Log` 注解使用，覆盖了：
- 系统管理模块（用户、角色、菜单、部门、岗位、字典、参数配置等）
- 资产管理模块（设备、配件、耗材、软件、故障、盘点等）
- 消息管理模块

### 1.3 菜单位置问题

操作日志菜单当前配置：
- 菜单ID: 500
- 菜单名称: 操作日志
- 父级菜单: 系统工具（parent_id=3）
- 排序号: 9（在系统工具的最后）

**问题**：操作日志放在"系统工具"目录下，位置较深，用户不容易找到。

---

## 二、优化方案

### 2.1 调整菜单位置

**目标**：将操作日志移到更显眼的位置

- 从"系统工具"移到"系统管理"目录下
- 调整排序号，放在合适的位置
- 图标调整为更符合日志语义的图标

### 2.2 操作日志页面优化

**目标**：提升操作日志页面的可读性和易用性

1. **优化列表展示**：
   - 操作时间（突出显示）
   - 操作人
   - 操作模块
   - 操作类型（新增/修改/删除/导出等，用不同颜色标签）
   - 操作状态（成功/失败）
   - 操作IP
   - 耗时

2. **增强搜索筛选**：
   - 按操作人搜索
   - 按操作模块筛选
   - 按操作类型筛选
   - 按状态筛选
   - 按时间范围筛选

3. **详情弹窗优化**：
   - 展示完整的请求参数
   - 展示响应结果
   - 展示异常信息（如果失败）

### 2.3 新增登录日志功能（可选）

**目标**：增加登录日志记录，追踪用户登录行为

- 新增 `sys_login_log` 表
- 记录登录时间、登录人、登录IP、登录状态、浏览器、操作系统等
- 前端增加登录日志页面

### 2.4 补充遗漏的操作日志

**目标**：确保所有重要操作都有日志记录

检查并补充以下模块的 `@Log` 注解：
- 资产分配/归还
- 资产报废/处置
- 消息发送
- 导入操作
- 等等

## 五、风险与注意事项

1. **菜单移动后权限问题**：确保admin角色有操作日志的权限
2. **数据库兼容性**：已部署的系统需要手动调整菜单或重新初始化
3. **日志量控制**：操作日志可能快速增长，需要考虑定期清理或归档策略
4. **性能影响**：AOP切面记录日志使用异步方式，不影响主业务性能
