## 一、环境准备

### 1.1 服务器要求

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB |
| 硬盘 | 20 GB | 50 GB |
| 系统 | Linux (CentOS 7+ / Ubuntu 20.04+) | Linux (CentOS 7+ / Ubuntu 20.04+) |

### 1.2 安装 Docker

```bash
# CentOS / RHEL
yum install -y yum-utils
yum-config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
yum install -y docker-ce docker-ce-cli containerd.io
systemctl start docker
systemctl enable docker

# Ubuntu / Debian
apt update
apt install -y apt-transport-https ca-certificates curl
curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | apt-key add -
add-apt-repository "deb [arch=amd64] https://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable"
apt install -y docker-ce docker-ce-cli containerd.io
systemctl start docker
systemctl enable docker
```

### 1.3 安装 Docker Compose

```bash
# 下载 docker-compose
curl -L "https://github.com/docker/compose/releases/download/v2.27.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 添加执行权限
chmod +x /usr/local/bin/docker-compose

# 验证
docker-compose --version
```

### 1.4 安装 Node.js 和 npm（运行脚本用）

```bash
# CentOS / RHEL
curl -fsSL https://rpm.nodesource.com/setup_20.x | bash -
yum install -y nodejs

# Ubuntu / Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 验证
node --version
npm --version
```

## 二、首次部署

### 2.1 上传代码到服务器

```bash
# 方式一：Git 拉取
git clone <仓库地址> /opt/leixi-manage
cd /opt/leixi-manage

# 方式二：SCP 上传（本地执行）
scp -r ./leixi-manage root@<服务器IP>:/opt/
```

### 2.2 修改环境配置

编辑 `.env` 文件：

```bash
vim .env
```

**必改项**：

```env
# 数据库密码（生产环境务必修改）
MYSQL_ROOT_PASSWORD=你的root密码
MYSQL_PASSWORD=你的用户密码

# 访问地址（服务器内网 IP 或 localhost）
DOMAIN_NAME=localhost
```

**端口配置说明**：

| 服务 | 配置项 | 默认值 | 说明 |
|------|--------|--------|------|
| MySQL | `MYSQL_HOST_PORT` | 3307 | 宿主机访问端口，避免与本地 MySQL 冲突 |
| 后端 | `APP_HOST_PORT` | 8088 | 后端 API 直接访问端口 |
| Nginx | `NGINX_HTTP_HOST_PORT` | 80 | 前端页面访问端口 |

> **注意**：`MYSQL_PORT`、`APP_PORT` 是容器内部端口，一般不需要修改。需要改外部端口时修改 `*_HOST_PORT` 即可。

### 2.3 启动服务

```bash
# 方式一：首次部署或完全重建（推荐）
npm run init

# 方式二：只启动服务（已有镜像时使用）
npm run docker:up

# 方式三：启动并构建镜像（代码有修改时使用）
npm run docker:up:build
```

**命令说明**：

| 命令 | 场景 | 是否安装依赖 | 是否构建镜像 | 是否初始化数据库 |
|------|------|-------------|-------------|-----------------|
| `npm run init` | 首次部署 | ✅ | ✅ | ✅ |
| `npm run docker:up` | 日常启动（镜像已存在） | ❌ | ❌ | ❌ |
| `npm run docker:up:build` | 代码有修改，需要重建 | ❌ | ✅ | ❌ |

> **提示**：首次部署后，以后启动只需 `npm run docker:up`，不需要每次都 `npm run init`！

启动过程（首次）：
1. 构建前端镜像（Node 构建 Vue 项目 → Nginx 静态文件）
2. 构建后端镜像（Maven 打包 Spring Boot）
3. 启动 MySQL 容器（自动执行 sql/ 下的初始化脚本）
4. 启动后端容器（等待 MySQL 就绪）
5. 启动 Nginx 容器（反向代理前端 + 后端 API）

### 2.4 查看启动状态

```bash
# 查看服务状态
npm run docker:status

# 实时查看日志
npm run docker:logs
```

看到类似输出表示启动成功：
```
ciyo-mysql   | ready for connections
ciyo-app     | Started CiyoAssetApiApplication in X.XXX seconds
ciyo-nginx   | start worker process
```

## 三、访问方式

### 3.1 架构说明

```
浏览器 → Nginx (80端口) ──┬── 前端静态文件
                         │
                         └── /api → Spring Boot (8088) → MySQL (3306)
```

### 3.2 服务器本机访问

浏览器打开：`http://localhost`

（如果修改了 `NGINX_HTTP_HOST_PORT`，访问 `http://localhost:<端口号>`）

### 3.3 局域网访问

1. 查看服务器内网 IP：
```bash
ip addr
```

2. 其他电脑浏览器打开：`http://<服务器IP>`

例如：`http://192.168.1.100`

### 3.4 使用内网域名访问（可选）

如果想使用域名 `manage.leixi.com` 访问：

**步骤 1**：修改 `.env`
```env
DOMAIN_NAME=manage.leixi.com
```

**步骤 2**：重启 nginx
```bash
npm run docker:nginx:build
```

**步骤 3**：每台要访问的电脑修改 hosts 文件

- Windows: `C:\Windows\System32\drivers\etc\hosts`
- Mac/Linux: `/etc/hosts`

添加一行：
```
192.168.1.100   manage.leixi.com
```

（把 IP 换成服务器实际内网 IP）

**步骤 4**：浏览器访问 `http://manage.leixi.com`

### 3.5 直接访问后端 API

如果需要直接调用后端接口（绕过 nginx）：

```
http://<服务器IP>:8088/api/...
```

### 3.6 直接连接 MySQL

使用数据库客户端工具连接：

| 配置项 | 值 |
|--------|-----|
| 主机 | localhost 或服务器 IP |
| 端口 | `MYSQL_HOST_PORT`（默认 3307） |
| 用户名 | `MYSQL_USER`（默认 ciyo） |
| 密码 | `MYSQL_PASSWORD`（默认 ciyo123456） |
| 数据库 | `MYSQL_DATABASE`（默认 ciyo_itam） |

## 四、修改代码后如何更新

### 4.1 重要说明

**Docker 部署是静态构建的，不支持代码热更新。** 修改代码后需要重新构建镜像并重启容器。

| 场景 | 是否实时更新 | 更新方式 |
|------|-------------|----------|
| 前端代码修改 | ❌ 不支持 | 重建前端镜像，重启 nginx 容器 |
| 后端代码修改 | ❌ 不支持 | 重建后端镜像，重启 app 容器 |
| 配置文件修改 | ❌ 不支持 | 重启对应容器 |
| nginx 配置修改 | ⚠️ 部分支持 | 重新加载 nginx 配置 |

### 4.2 前端代码更新

修改了 `itasset-front/` 下的代码：

```bash
# 方式一：一键更新（推荐）
npm run docker:nginx:build

# 方式二：手动步骤
docker-compose build nginx    # 重新构建前端镜像
docker-compose up -d nginx    # 重启 nginx 容器
```

更新时间：约 2-5 分钟（取决于构建速度）

### 4.3 后端代码更新

修改了 `ciyo-*/` 下的 Java 代码：

```bash
# 方式一：一键更新
docker-compose up -d --build app

# 方式二：手动步骤
docker-compose build app      # 重新构建后端镜像
docker-compose up -d app      # 重启 app 容器
```

更新时间：约 5-10 分钟（Maven 打包较慢）

### 4.4 前后端都修改了

```bash
# 全部重建并重启
npm run docker:rebuild
```

### 4.5 仅修改 nginx 配置

修改了 `nginx/nginx.conf`：

```bash
# 方式一：热重载（不中断服务）
npm run docker:nginx:reload

# 方式二：重启 nginx
docker-compose restart nginx
```

### 4.6 仅修改 .env 配置

修改了 `.env` 文件：

```bash
# 重启所有服务使配置生效
npm run docker:restart

# 或者只重启某个服务
docker-compose restart nginx
docker-compose restart app
```

### 4.7 本地开发模式（支持热更新）

如果需要频繁修改代码，建议使用本地开发模式：

```bash
# 启动前端开发服务器（修改代码自动刷新）
npm run dev:front

# 启动后端开发服务器（需要 IDE 支持热部署）
npm run dev
```

前端开发服务器默认地址：`http://localhost:5173`

## 五、常用命令速查

### 5.1 Docker 完整环境

```bash
# 启动所有服务
npm run docker:up

# 构建并启动
npm run docker:up:build

# 停止所有服务
npm run docker:down

# 重启所有服务
npm run docker:restart

# 查看日志
npm run docker:logs

# 查看服务状态
npm run docker:status

# 完全清理（删除数据卷！谨慎使用）
npm run docker:clean

# 重新构建并启动
npm run docker:rebuild
```

### 5.2 单服务日志

```bash
# 查看后端日志
npm run docker:logs:app

# 查看 MySQL 日志
npm run docker:logs:mysql

# 查看 Nginx 日志
npm run docker:logs:nginx
```

### 5.3 Nginx 相关

```bash
# 启动 nginx
npm run docker:nginx

# 重新构建并启动 nginx
npm run docker:nginx:build

# 热重载 nginx 配置
npm run docker:nginx:reload

# 重启 nginx
npm run docker:nginx:restart
```

### 5.4 数据库相关

```bash
# 进入 MySQL 命令行
npm run docker:mysql

# 备份数据库
docker exec ciyo-mysql mysqldump -uciyo -pciyo123456 ciyo_itam > backup_$(date +%Y%m%d).sql

# 恢复数据库
docker exec -i ciyo-mysql mysql -uciyo -pciyo123456 ciyo_itam < backup.sql
```

### 5.5 进入容器

```bash
# 进入后端容器
npm run docker:shell:app

# 进入 MySQL 容器
npm run docker:shell:mysql

# 进入 Nginx 容器
npm run docker:shell:nginx
```

## 六、常见问题

### 6.1 启动失败，端口被占用

```bash
# 查看端口占用
netstat -tlnp | grep 80
netstat -tlnp | grep 3306
netstat -tlnp | grep 8088

# 修改 .env 中的宿主机端口
MYSQL_HOST_PORT=3307      # MySQL 外部端口
APP_HOST_PORT=8089        # 后端外部端口
NGINX_HTTP_HOST_PORT=8080 # Nginx 外部端口
```

改完后重启：
```bash
npm run docker:up
```

### 6.2 数据库初始化失败

```bash
# 查看 MySQL 日志
npm run docker:logs:mysql

# 常见原因：
# 1. 数据卷已存在旧数据 - 删除数据卷后重启
npm run docker:clean
npm run docker:up

# 2. SQL 文件有错误 - 检查 sql/ 目录下的 SQL 文件
```

### 6.3 前端页面空白

```bash
# 查看 nginx 日志
npm run docker:logs:nginx

# 检查前端是否构建成功
docker exec ciyo-nginx ls /usr/share/nginx/html

# 重新构建前端
npm run docker:nginx:build
```

### 6.4 后端连不上数据库

```bash
# 检查 MySQL 是否正常运行
npm run docker:status

# 检查数据库连接配置
docker exec ciyo-app env | grep MYSQL

# 手动测试数据库连接
npm run docker:mysql
```

### 6.5 Nginx 报 502 Bad Gateway

这表示 nginx 连不上后端服务：

```bash
# 检查后端是否启动成功
npm run docker:logs:app

# 查看后端状态
npm run docker:status

# 如果后端还在启动中，等待一会儿即可
```

### 6.6 清理所有数据重新部署

⚠️ 注意：这会删除所有数据库数据

```bash
npm run docker:clean
npm run docker:up:build
```

### 6.7 如何升级系统

```bash
# 1. 拉取最新代码
git pull

# 2. 备份数据库
docker exec ciyo-mysql mysqldump -uciyo -pciyo123456 ciyo_itam > backup_$(date +%Y%m%d).sql

# 3. 重新构建并启动
npm run docker:rebuild
```

## 附录：文件结构说明

```
leixi-manage/
├── .env                    # 环境变量配置（端口、密码、域名等）
├── .dockerignore           # Docker 构建忽略文件
├── Dockerfile              # 后端 Dockerfile
├── Dockerfile.front        # 前端 Dockerfile (nginx)
├── docker-compose.yml      # Docker Compose 配置
├── docker-entrypoint.sh    # 后端启动脚本
├── package.json            # 快捷命令
├── 部署.md                 # 本文档
├── nginx/
│   └── nginx.conf          # Nginx 配置
├── sql/
│   ├── ciyo-itam.sql       # 主初始化脚本
│   └── update_0.2.sql      # 更新脚本
├── itasset-front/          # 前端代码
└── ciyo-*/                 # 后端代码
```

### 端口对照表

| 服务 | 容器名称 | 容器内部端口 | 宿主机映射端口（默认） | 说明 |
|------|----------|-------------|----------------------|------|
| Nginx | ciyo-nginx | 80 | 80 | 前端页面访问入口 |
| Spring Boot | ciyo-app | 8088 | 8088 | 后端 API 服务 |
| MySQL | ciyo-mysql | 3306 | 3307 | 数据库服务 |

> **提示**：所有宿主机端口都可以在 `.env` 文件中通过 `*_HOST_PORT` 配置修改。
